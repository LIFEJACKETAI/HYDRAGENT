     1→# HYDRAGENT
     2→
     3→**Embeddable AI Appointment Agent for Any Business**
     4→
     5→HYDRAGENT is a full-stack admin dashboard and embeddable chat widget that lets appointment-based businesses — restaurants, doctors, dentists, car mechanics, salons, spas, gyms, legal offices, and more — deploy an AI-powered agent on their website. The agent answers customer questions, books appointments, sends emails, and handles phone calls using a knowledge base you upload.

    40→## Features
    41→
    42→### Admin Dashboard (10 Sections)
    43→
    44→| Section | Description |
    45→|---------|-------------|
    46→| **Dashboard** | Overview stats — appointments, emails, calls, recent activity feed, upcoming bookings |
    47→| **Analytics** | 14-day booking trends, status breakdowns, top services, peak hours, day-of-week heatmaps, channel usage, system health |
    48→| **Knowledge Base** | Upload documents (text, markdown, files) or paste content directly. Toggle documents active/inactive to control what the agent knows |
    49→| **Appointments** | Full CRUD — create, confirm, complete, cancel. Filter by status. Duration tracking |
    50→| **Emails** | Inbox/sent/drafts tabs, compose emails, read full threads |
    51→| **Calls** | Log inbound/outbound calls, track duration, status (completed/missed/voicemail), quick stats |
    52→| **Integrations** | Connect Google Calendar, Outlook, SendGrid, Twilio, Calendly, Zapier |
    53→| **Embed Widget** | Live widget preview, customize colors/position/greeting, copy embed code |
    54→| **Agent Preview** | Test the AI agent in a split chat interface with live knowledge context panel |
    55→| **Admin & Settings** | Business profile editing, agent behavior config, database maintenance, data export, system info, danger zone |
    56→
    57→### AI Agent
    58→
    59→- Powered by LLM via `z-ai-web-dev-sdk`
    60→- Uses all active knowledge documents as context
    61→- Remembers conversation history (last 20 messages)
    62→- Configurable tone (Professional / Friendly / Casual / Formal)
    63→- Configurable response length (Short / Medium / Detailed)
    64→- Optional auto-confirmation of appointments
    65→- Optional customer info collection
    66→
    67→### Multi-Tenant Ready
    68→
    69→The admin panel is designed so you can reconfigure the entire business identity from **Settings** — change the name, type, phone, email, address, hours, description, and agent personality. The upload limit is set to **900 KB** per request.

   210→## Database Setup
   211→
   212→HYDRAGENT uses **SQLite** via Prisma ORM. The database file lives at `db/custom.db`.
   213→
   214→### Schema Overview (7 Models)
   215→
   216→| Model | Purpose |
   217→|-------|---------|
   218→| `Business` | Business name, type, contact info, widget customization |
   219→| `KnowledgeDoc` | Uploaded knowledge documents (text content, file type, active toggle) |
   220→| `Appointment` | Customer bookings with status workflow |
   221→| `EmailRecord` | Inbound and outbound emails |
   222→| `CallLog` | Phone call records with duration and status |
   223→| `Integration` | Third-party service connections (Google Calendar, Twilio, etc.) |
   224→| `ChatMessage` | Agent chat history |
   225→
   226→### Commands
   227→
   228→```bash
   229→# Push schema changes to the database (creates tables)
   230→bun run db:push
   231→
   232→# Generate the Prisma client
   233→bun run db:generate
   234→
   235→# Reset the database completely
   236→bun run db:reset
   237→```

   241→## Production Deployment
   242→
   243→### Option A: Docker (Recommended)
   244→
   245→Create a `Dockerfile` in the project root:
   246→
   247→```dockerfile
   248→FROM oven/bun:1 AS base
   249→WORKDIR /app
   250→
   251→# Install dependencies
   252→FROM base AS deps
   253→COPY package.json bun.lock ./
   254→RUN bun install --frozen-lockfile --production
   255→
   256→# Build the application
   257→FROM base AS builder
   258→COPY --from=deps /app/node_modules ./node_modules
   259→COPY . .
   260→RUN bun run build
   261→
   262→# Production image
   263→FROM base AS runner
   264→ENV NODE_ENV=production
   265→
   266→COPY --from=builder /app/.next/standalone ./
   267→COPY --from=builder /app/.next/static ./.next/static
   268→COPY --from=builder /app/public ./public
   269→COPY --from=builder /app/db ./db
   270→COPY --from=builder /app/Caddyfile ./Caddyfile
   271→
   272→EXPOSE 3000 81
   273→
   274→CMD ["sh", "-c", "caddy run --config Caddyfile --adapter caddyfile & bun .next/standalone/server.js"]
   275→```
   276→
   277→Build and run:
   278→
   279→```bash
   280→docker build -t hydragent .
   281→docker run -d -p 81:81 -p 3000:3000 \
   282→  -v hydragent-data:/app/db \
   283→  --name hydragent \
   284→  hydragent
   285→```
   286→

   288→
   289→```bash
   290→# 1. SSH into your server
   291→ssh user@your-server-ip
   292→
   293→# 2. Install Bun
   294→curl -fsSL https://bun.sh/install | bash
   295→
   296→# 3. Clone the repo
   297→git clone https://github.com/LIFEJACKETAI/HYDRAGENT.git
   298→cd HYDRAGENT
   299→

   301→bun install
   302→
   303→# 5. Set up environment

   305→nano .env  # Edit with your values

   308→bun run db:push
   309→
   310→# 7. Build for production
   311→bun run build
   312→

   314→caddy run --config Caddyfile --adapter caddyfile &
   315→NODE_ENV=production bun .next/standalone/server.js

   319→
   320→```bash
   321→sudo tee /etc/systemd/system/hydragent.service << 'EOF'
   322→[Unit]
   323→Description=HYDRAGENT AI Agent
   324→After=network.target
   325→
   326→[Service]
   327→Type=simple
   328→User=www-data
   329→WorkingDirectory=/var/www/hydragent
   330→Environment=NODE_ENV=production
   331→Environment=DATABASE_URL=file:/var/www/hydragent/db/custom.db
   332→ExecStart=/root/.bun/bin/bun .next/standalone/server.js

   334→RestartSec=5

   364→## Remote Client Setup
   365→
   366→This section covers how to deploy HYDRAGENT on a **client's machine** remotely — for example, setting up a dentist's office server from your own machine.
   367→
   368→### Step 1: Prepare the Machine
   369→
   370→The client's machine needs:
   371→- **OS:** Ubuntu 22.04+ / Debian 12+ / CentOS 8+ (or any Linux with Bun support)
   372→- **Minimum specs:** 1 CPU core, 1 GB RAM, 10 GB disk
   373→- **Recommended:** 2 CPU cores, 2 GB RAM, 20 GB disk
   374→- **Ports:** 80 (HTTP) and 443 (HTTPS) open, or any port you configure
   375→
   376→### Step 2: Transfer the Project
   377→
   378→**Option A: Git (recommended if the client has internet)**
   379→
   380→```bash
   381→# From YOUR machine — SSH into the client's server
   382→ssh user@client-server-ip
   383→
   384→# On the client's server:
   385→git clone https://github.com/LIFEJACKETAI/HYDRAGENT.git /opt/hydragent
   386→cd /opt/hydragent
   387→```
   388→
   389→**Option B: SCP (copy files directly from your machine)**
   390→
   391→```bash
   392→# From YOUR machine — copy the project directory
   393→scp -r ./HYDRAGENT user@client-server-ip:/opt/hydragent
   394→
   395→# Then SSH in
   396→ssh user@client-server-ip
   397→cd /opt/hydragent
   398→```
   399→
   400→**Option C: rsync (faster for large projects, supports resume)**
   401→
   402→```bash
   403→# From YOUR machine
   404→rsync -avz --progress ./HYDRAGENT/ user@client-server-ip:/opt/hydragent/
   405→```
   406→
   407→### Step 3: Install & Build
   408→
   409→SSH into the client's machine and run:
   410→
   411→```bash
   412→# Install Bun (if not already installed)
   413→curl -fsSL https://bun.sh/install | bash
   414→source ~/.bashrc
   415→
   416→# Navigate to the project
   417→cd /opt/hydragent
   418→
   419→# Install dependencies
   420→bun install
   421→
   422→# Create the environment file
   423→cat > .env << 'ENVEOF'
   424→DATABASE_URL="file:/opt/hydragent/db/custom.db"
   425→PORT=3000
   426→HOSTNAME=0.0.0.0
   427→ENVEOF
   428→
   429→# Create the database directory
   430→mkdir -p db
   431→
   432→# Push the database schema
   433→bun run db:push
   434→
   435→# Build for production
   436→bun run build
   437→```
   438→
   439→### Step 4: Configure the Business
   440→
   441→Before starting, configure the business details. You have two options:
   442→
   443→**Option A: Use the Admin Panel (GUI — recommended)**
   444→
   445→1. Start the dev server temporarily:
   446→   ```bash
   447→   bun run dev &
   448→   ```
   449→2. Open `http://client-server-ip:3000` in your browser
   450→3. Navigate to **Admin & Settings** in the sidebar
   451→4. Fill in the business name, type, phone, email, address, hours, and description
   452→5. Upload knowledge documents in the **Knowledge Base** section
   453→6. Stop the dev server: `kill %1`
   454→
   455→**Option B: Use the Seed API (quick demo data)**
   456→
   457→```bash
   458→# Start the server, then seed sample data
   459→bun run dev &
   460→sleep 5
   461→curl -X POST http://localhost:3000/api/seed
   462→```
   463→

   465→
   466→```bash
   467→# Start the server first, then:

   470→  -d '{

   481→    "widgetPosition": "bottom-right",

   496→### Step 5: Start the Service
   497→
   498→**For development/testing:**
   499→
   500→```bash
   501→cd /opt/hydragent
   502→bun run dev
   503→```
   504→
   505→**For production (with Caddy reverse proxy):**
   506→
   507→```bash
   508→cd /opt/hydragent
   509→
   510→# Install Caddy if not present
   511→sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https
   512→curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
   513→curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
   514→sudo apt update && sudo apt install caddy
   515→
   516→# Start both Caddy and the app
   517→nohup caddy run --config Caddyfile --adapter caddyfile > /var/log/caddy.log 2>&1 &
   518→NODE_ENV=production nohup bun .next/standalone/server.js > /var/log/hydragent.log 2>&1 &
   519→```
   520→
   521→**As a systemd service (auto-restarts on crash/reboot):**
   522→
   523→```bash
   524→sudo tee /etc/systemd/system/hydragent.service << 'EOF'
   525→[Unit]
   526→Description=HYDRAGENT AI Appointment Agent
   527→After=network.target
   528→
   529→[Service]
   530→Type=simple
   531→User=root
   532→WorkingDirectory=/opt/hydragent
   533→Environment=NODE_ENV=production
   534→Environment=DATABASE_URL=file:/opt/hydragent/db/custom.db
   535→ExecStart=/root/.bun/bin/bun /opt/hydragent/.next/standalone/server.js
   536→Restart=always
   537→RestartSec=5
   538→StandardOutput=append:/var/log/hydragent.log
   539→StandardError=append:/var/log/hydragent.log
   540→
   541→[Install]
   542→WantedBy=multi-user.target
   543→EOF
   544→
   545→sudo systemctl daemon-reload
   546→sudo systemctl enable hydragent
   547→sudo systemctl start hydragent
   548→
   549→# Check status
   550→sudo systemctl status hydragent
   551→
   552→# View logs
   553→journalctl -u hydragent -f
   554→```

   836→## Troubleshooting
   837→
   838→### White screen on load
   839→- Ensure the database exists: `ls -la db/custom.db`
   840→- Check the dev server log: `tail -20 dev.log`
   841→- Verify dependencies are installed: `bun install`
   842→
   843→### Upload fails for large files
   844→- The upload limit is set to 900 KB in `next.config.ts` (`serverBodySizeLimit: 900000`)
   845→- Increase it if needed, but be mindful of your server's memory
   846→
   847→### Database errors
   848→- Run `bun run db:push` to re-sync the schema
   849→- If corrupted, delete `db/custom.db` and run `bun run db:push` again
   850→- Seed fresh data: `curl -X POST http://localhost:3000/api/seed`
   851→
   852→### Port already in use
   853→- Kill the existing process: `lsof -ti:3000 | xargs kill -9`
   854→- Or change the port in `.env`: `PORT=3001`
   855→
   856→### AI chat not responding
   857→- Ensure `z-ai-web-dev-sdk` is installed: `bun install`
   858→- Check that knowledge documents are active in the Knowledge Base
   859→- Check the server console for SDK errors
